# Plan: ACT Living Website Build Reset

> Slug: `act-living-website-build-reset`
> Created: 2026-04-11
> Status: in-progress
> Owner: Codex + Ben

## Objective

Reset the work around the real goal: build ACT's public websites from the wiki as the canonical memory layer, with Empathy Ledger providing live editorial/media overlays, and the website composing those two cleanly. The point is to stop spending cycles on unrelated cleanup and instead make adding or updating a project, work, story, or article feel routine, durable, and syndication-ready across the ACT hub and project spokes.

## Task Ledger

- [ ] Freeze side-quest work unless it directly unblocks `wiki -> sync -> website`
- [ ] Normalize the wiki project model so every real project/work page has canonical frontmatter and cross-system identity
- [ ] Reduce the website data contract to two upstream inputs only: wiki truth and EL editorial/media
- [ ] Make the ACT hub render all flagship projects and works from those generated snapshots, not hand-maintained copy
- [ ] Define the hub/spoke syndication rules for project sites, works, stories, and articles
- [ ] Add wiki/website lint checks so drift is visible before more UI work happens
- [ ] Ship one documented operating workflow for `new project`, `new work`, `new story`, and `new article`
