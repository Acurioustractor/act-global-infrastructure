## What's missing in the system

### Reports + acquittals delivered vs owed

There's no current view of "reports we've sent to funders vs reports overdue." This is the biggest gap. Each paid supporter likely has an acquittal/reporting commitment that needs tracking. Suggested addition:

```sql
-- New table: funder_reports
CREATE TABLE funder_reports (
  funder_slug TEXT,
  report_name TEXT,
  due_date DATE,
  delivered_date DATE,
  status TEXT,  -- 'due-soon' | 'overdue' | 'delivered'
  project_code TEXT,
  url TEXT
);
```

Already partially captured in `wiki/narrative/funders.json` via `next_report_due` + `next_report_name` fields (added in commit `0d05794` on 2026-05-22). Should sync to this table for queryability.

### Updates we've developed vs need to develop

For each active supporter, list:
- Public-consent stories ready to share (from Empathy Ledger)
- Project progress data (from cross-codebase feed)
- Financial position (from this view)

This is what the newsletter pipeline is now positioned to surface. With the 9-funder roster locked in (`d371425`), per-recipient drafts auto-pull from these sources.

Filter: Outstanding > $0, sorted by aging. The "must call this week" list.
