# A — Finance money-state & reporting

> Data-lineage audit of every command-center finance money-state + reporting surface.
> Read-only. Verdict legend per `_schema-truth.md` (🟢 real · 🟡 stale · 🟠 misleading · 🔴 broken · ⚫ not-wired · ⚠️ unverified).
> All table/column/view existence facts verified against shared DB `tednluwflfhxyucgwigh` (5 queries run; rest from `_schema-truth.md`).

## Top findings

- **There are ~9 distinct, mutually-inconsistent implementations of "expense / income / P&L".** No single source of truth exists. The shared `lib/finance/` only holds *pure math* (runway, aging, overhead) — it does NOT define how income/expense are extracted, so every route re-invents that, and they disagree:
  1. **`api/finance/projects/[code]` `realExpenseRows`** — the GOOD one: deduped ACCPAY bills + unmatched SPEND/SPEND-OVERPAYMENT by `type`. **Reference implementation.**
  2. **`api/finance/projects/[code]/transactions`** — same good dedup logic (bills + SPEND, mirror-payment detection). Consistent with #1.
  3. **`v_project_money_state` view** (used by `/finance/command`) — correct `type`-based SPEND/RECEIVE, but **bank-txn only, ignores ACCPAY bills** → understates expenses.
  4. **`project_monthly_financials` cache** (used by `overview`, `projects` list, `accountant-pack`) — pre-aggregated, **bank-txn only, no bills**, stops before current Xero snapshot.
  5. **`api/finance/revenue-reality`** — `type IN ('RECEIVE','RECEIVE-TRANSFER')` / `('SPEND','SPEND-TRANSFER')` → **counts transfers as income/expense** (schema-truth says exclude `%-TRANSFER`).
  6. **`api/finance/rd-tracking` + `api/finance/projects` rdSpend + `accountant-pack` R&D** — `total < 0` → **the SIGN BUG** (SPEND stored positive) → near-zero R&D.
  7. **`api/finance/rd-dashboard`** — `rd_eligible=true` + `type IN ('SPEND','SPEND-TRANSFER')` → transfers leak; different R&D project list again.
  8. **`reports/{monthly,yearly,weekly}` + `reports/profit-loss` + `api/revenue-streams`** — `type='RECEIVE'/'SPEND'` with `Math.abs`, **bank-txn only, no bills, no two-account scope**.
  9. **`reports/cash-flow-forecast`** — "current balance" = receivables − payables (an AR position, **not cash**).
- **Trustworthy money surfaces (use these):** `api/finance/projects/[code]` and `…/[code]/transactions` (deduped bills+spend, Harvest stage budget, audit alerts) and the `v_project_money_state` view behind `/finance/command` (correct sign, but bills-blind). Everything else is misleading, stale, or broken.
- **The SIGN BUG is still live in 3+ finance routes:** `rd-tracking`, `projects` (list) rdSpend, `accountant-pack` R&D all use `total < 0` → silent ~$0. (Same bug the /company audit flagged; the project-detail route is the only one that fixed it.)
- **No route applies the two-account rule.** `overview` and `command` sum ALL active/non-archived bank accounts for cash (includes NM Personal + ACT Maximiser). Real ACT cash = NAB Visa #8815 + ACT Everyday only.
- **Dead-table routes returning silent zeros / 500s:** `api/debt` (debts/debt_payments/debt_scenarios/properties — all dead → 500); `api/finance/cashflow-explained` (`v_cashflow_explained` dead → 500); `api/finance/tax` donations leg + `api/revenue-streams` donations leg (`donations` dead → 0); `reports/monthly` + `reports/yearly` (`contacts`/`communications` dead → 0, plus `project_knowledge.type` wrong column → 0 knowledge).
- **Accountant-facing artefact is the highest-risk surface.** `api/finance/accountant-pack` (downloadable HTML for an accountant) has: SIGN-BUG-free but **bills-blind P&L**, stale `[ACT-EL,IN,JH,GD]` R&D list, **hardcoded wrong "AusIndustry by April 30 2026" deadline**, wrong runway formula, and labels itself **"ACT Foundation" + sole-trader ABN**. `api/finance/tax` hardcodes the same wrong entity/ABN list.
- **Hardcoded wrong R&D deadlines persist** in `rd-tracking` ("FY2024-25 R&D Registration 2026-04-30") and `accountant-pack`. Per MEMORY, FY24-25 is forfeited and 30 Apr 2026 does NOT apply to ACT.
- **`project_budgets` column drift breaks every budget %.** `overview`, `projects` list, and the `[code]` route read `annual_budget`/`ytd_budget`/`fy` — real columns are `budget_amount`/`budget_type`/`fy_year`/`month`. Budget utilisation is silently null everywhere except ACT-HV (which uses the hardcoded `HARVEST_ZONES` config instead).
- **`reports/page.tsx` ↔ `/api/reports/weekly` contract mismatch:** page expects `data.sections.financial.cashPosition`; route returns `{finance:{income,expenses,net}}`. Shapes don't match → page renders empty.
- **Stale/abandoned feeds:** `financial_snapshots` (20d stale, 17 rows) drives `runway`, `cashflow`, `revenue-streams` balances; `fundraising_pipeline` (abandoned 2.5mo) drives `runway` fundraising leg.

## What's salvageable

**The real money story (keep + build on):**
- `api/finance/projects/[code]` (`realExpenseRows`) and `…/[code]/transactions` — the only routes that compute expense correctly (deduped ACCPAY bills + unmatched SPEND/SPEND-OVERPAYMENT, by `type`). This is the reference logic the /company audit pointed at.
- `v_project_money_state` view — sign-correct income/expense; the cleanest aggregate, just needs ACCPAY bills folded in and a two-account cash join.
- `lifetime-ledger` — clean ACCREC customer book.
- `data-quality`, `today-actions`, `actions`, `pilot-risks`, `ack*`, `pty-readiness`, `command-delta` — operational queues, honest, no money-truth risk.

**Duplicate/contradict → fold into one:**
- **Expense/P&L:** collapse `overview`, `projects` (list), `accountant-pack`, `revenue-reality`, `reports/{monthly,yearly,weekly,profit-loss}`, `money-alignment` allocation onto a single `lib/finance` extractor built from `realExpenseRows` (deduped bills+spend, `type`-based, two-account scope, Jan-2026 Harvest cutoff). They currently disagree because each re-derives expense (bills-blind cache, sign bug, transfer leakage).
- **Cash + runway:** `overview`, `command`, `runway`, `cashflow`, `cashflow-explained`, `reports/cash-flow-forecast` each compute "cash" differently (all-account sum / non-archived sum / stale snapshot / AR−AP / dead view). Pick one cash function: `xero_bank_accounts.current_balance` for NAB Visa #8815 + ACT Everyday only; runway = that ÷ deduped monthly burn.
- **R&D:** `rd-tracking` (sign bug), `rd-dashboard` (transfers + list `EL,JH,IN,DO`), `accountant-pack` (list `EL,IN,JH,GD`), `projects` list rdSpend (sign bug) — four schemes, three project lists, two sign behaviours. Unify the R&D project list + the deduped bill-aware spend extractor, and remove every hardcoded "30 Apr 2026" deadline (FY24-25 forfeited; FY25-26 lodges Jul 2026–30 Apr 2027).

**Decommission or rebuild (currently broken/zero):**
- `api/debt` (4 dead tables → 500), `cashflow-explained` (`v_cashflow_explained` dead), the `donations` legs in `tax` + `revenue-streams`, the `contacts`/`communications` + `project_knowledge.type` legs in `reports/{monthly,yearly,weekly}`, and the `reports/page.tsx ↔ /api/reports/weekly` shape mismatch.

**Highest-urgency fix:** `accountant-pack` + `tax` — they carry the wrong legal entity ("ACT Foundation" / sole-trader ABN / "A Kind Tractor"), a wrong R&D deadline, and bills-blind numbers into an accountant-facing artefact.
