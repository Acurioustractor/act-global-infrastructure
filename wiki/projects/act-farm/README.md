# ACT Farm

> *Regenerative agriculture, creative practice, therapeutic landscapes — on 150 acres of Jinibara Country.*

The farm cluster has two files because the work has two layers: the **program** (act-farm.md, the methodology and the practice) and the **place** (black-cockatoo-valley.md, the 150-acre property where the program lives). Both belong in the same folder because neither one stands alone — the program needs the land, and the land needs the program.

The canonical title for the cluster is **act-farm** (matching the project filename).

## System Position

This cluster is the durable memory for ACT's land-practice program.

- the wiki explains the method, the place, and the relationship between ACT Farm and [[black-cockatoo-valley|Black Cockatoo Valley]]
- [[empathy-ledger|Empathy Ledger]] should carry live field proof: photos, drone footage, voice, seasonal notes, and any media-led editorial choices
- the websites should compose those layers into `/farm`, the BCV project page, and any future place-led public surfaces

The important rule is that the land story is written once in the wiki, then refreshed publicly through EL and the sites. The contract is in [[wiki-project-and-work-sync-contract|Wiki Project & Work Sync Contract]] and [[living-website-operating-system|Living Website Operating System]].

## Why two files, not one

Two separate files are kept (rather than merged) because:
1. The program (act-farm) can outlive any single property — the methodology travels
2. The property (BCV) has its own history as Jinibara Country independent of the program
3. Future ACT Farm satellites at other properties would file under `act-farm.md` while keeping `black-cockatoo-valley.md` as the founding-place reference
