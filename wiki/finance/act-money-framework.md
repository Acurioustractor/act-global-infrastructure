## Section D — Decision Tree (the simple algorithm)

> **"How do we move $X to founders this month?"**

```
1. Is the Pty cash-positive after tax reserve + 3-month float?
   NO → no draw beyond salary; raise cash or pause
   YES → continue

2. Is salary ($10K/mo each) already running (Channel 8)?
   NO → fix that first; cheapest channel
   YES → continue

3. Is there work Ben did that's genuine photo/storytelling for ACT?
   YES → invoice via Knight Photography (Channel 13)
   NO  → skip

4. Is it month 12 (June) with retained profits?
   NO → Director loan (Channel 11) — but track for EOY settle
   YES → continue

5. EOY: declare franked dividend → trust → distribute
   - Standard route → Channel 14 → 16 (to Ben/Nic at marginal)
   - Above marginal sweet spot? → Channel 18 (to bucket company at 25%)
   - Spouse / adult child available + deed permits? → Channel 17 (watch s100A)
   - Capital gain in mix? → Channel 21 (stream the gain separately)
```

That's the whole algorithm. Five steps.

### E1. Position now for Small Business CGT concessions on CivicGraph exit

**Opportunity:** material tax saving on a CivicGraph exit (Y5 target $4-10M EV per five-year-cashflow-model) **if** structured correctly.

**Critical structural choice — share sale vs asset sale.** The CGT-concession math depends on which.

#### Scenario A — Share sale (trusts sell their shares in Pty)

```
Sale price (shares)              $5,000,000
Cost base (founder shares)          $50,000
Capital gain                     $4,950,000

Concessions stack (subject to eligibility):
  50% active asset reduction    -$2,475,000  (gain → $2,475,000)
  General 50% CGT discount      -$1,237,500  (trust held >12mo) → $1,237,500
  Retirement exemption          -$1,000,000  (Ben + Nic, $500K lifetime each)
                                  --------
Taxable gain (split between 2)     $237,500
Tax payable per founder ~47%        $55,800

NET PER FOUNDER ~$2.4M of $2.5M share
```

#### Scenario B — Asset sale (Pty sells the CivicGraph IP/business)

```
Sale price (assets)              $5,000,000
Cost base                           $50,000
Capital gain (in Pty)            $4,950,000

Concessions stack:
  50% active asset reduction    -$2,475,000
  ✗ NO general 50% CGT discount  (companies don't get it)
  Retirement exemption*         -$1,000,000  (only via specific company-payment-to-stakeholder mechanics)
                                  --------
Taxable gain in Pty              $1,475,000
Tax at 25% company rate            $368,750

Then to get cash to founders: dividend → trust → individual at marginal rate (with franking)
```

**Implication [I]:** share sale is materially more tax-efficient than asset sale. Standard Ledger should pressure-test which is achievable for CivicGraph exit — and consider carving CivicGraph into its own Pty pre-exit for cleaner share-sale optics.

**Active-asset test [U]:** half-period test means CivicGraph IP must have been an active asset for at least half the ownership period. If 328-G rollover at cutover preserves the sole trader's prior active-asset history, the clock is favourable. **Confirm with Standard Ledger.**

**MNAV $6M test [V]:** aggregates connected entities + affiliates (trusts, Knight Photography, Harvest, Farm). Track ACT Pty net asset value annually. By Y5, the $6M test will be the gating concession qualifier — if approaching, may need to use $2M aggregated turnover test instead (probably fails by Y5 unless CivicGraph is carved out).

**Action now:**
1. Confirm CivicGraph activity is run inside ACT Pty (not separate entity that hasn't established active-asset history).
2. Standard Ledger documents active-asset usage — annual evidence CivicGraph IP is "used in carrying on a business."
3. Track combined net asset value across connected entities annually.
4. Consider carving CivicGraph into its own Pty (~Y3-4, pre-exit) for cleaner share-sale structure.
5. Decide which trust beneficiaries will use their $500K lifetime retirement exemption — once used, can't be re-used.

### E3. Subdivision 328-G rollover for cutover asset transfer

**Opportunity:** defer all asset-related tax events at cutover, including IP, equipment, depreciating assets, and trading stock — areas where Subdivision 122-A would leave gaps.

**Action [V]:** instruct Standard Ledger to use **Subdivision 328-G** (small business restructure rollover), not 122-A. 328-G covers all asset types.

**Eligibility check [V]:**
- Both entities small business entities (aggregated turnover < $10M) — ✅ ACT well under
- "Genuine restructure" of business — ✅ Nic's sole trader → Pty meets this
- Same ultimate economic ownership before and after — ✅ Nic remains beneficial owner via Marchesi Trust
- Australian residents — ✅
- Election made by both parties before/at cutover — needs to be done formally

**Important [V]:** the election is **Nic's sole trader → ACT Pty**. Nic signs as transferor; ACT Pty signs as transferee. Knight Photography is **not** part of this rollover (it stays running, separate ABN, no asset transfer).

**Risk if missed:** asset transfers at market value trigger CGT events. Substantial IP (codebases, methodologies, trademarks built up in Nic's sole trader during R&D work) could be valued at $200K-$500K+, generating significant tax cost.

## Section L — Notion surface (page + database)

Two complementary Notion surfaces, both auto-populated from Supabase:

### L1. ACT Money Framework page (exec view)

Glanceable one-screen summary: bank position, top opportunities, compliance flags, pile mix, plus a pile-by-pile drill-down with deep links to Xero / GHL / funder URLs.

```bash
# Update the Notion exec page
node scripts/sync-money-framework-to-notion.mjs

# Local markdown deep-dive (9 panels — auto-generated)
node scripts/sync-money-framework-to-notion.mjs --markdown
```

Notion page ID: `moneyFramework` in `config/notion-database-ids.json`. Pile mapping lives in `config/pile-mapping.json` (single source of truth for the script + backfill).

### L2. ACT Opportunities database (operational view)

A real Notion **database** of every opportunity, with schema:
- **Name** (title)
- **Pile** (select: Voice/Flow/Ground/Grants/Other)
- **Source** (select: GHL/Xero/Foundation)
- **Stage** (select: Open/Outstanding/Overdue/Won/Lost/Discovered/Researching/Archived)
- **Amount** (AUD)
- **Deadline** (date)
- **Source URL** (clickable — Xero invoice / GHL deal / grant URL)
- **Project Code** / **Funder/Contact** / **Pipeline** / **External ID** / **Last Synced**

Rows are upserted from:
- GHL open + won + lost opportunities (315)
- Xero outstanding receivables (~30)
- Foundation grants $50K-$5M with deadline open (~50)

Notion lets you build views: by Pile (kanban), by Stage (kanban), by Source, table sorted by Amount, etc. Each row links to the source system for action.

```bash
# Sync all sources
node scripts/sync-opportunities-to-notion-db.mjs

# Single source
node scripts/sync-opportunities-to-notion-db.mjs --source ghl
node scripts/sync-opportunities-to-notion-db.mjs --source xero
node scripts/sync-opportunities-to-notion-db.mjs --source foundation
```

Database ID: `opportunitiesDb` and `opportunitiesDataSource` in `config/notion-database-ids.json`. Stale rows (in DB but no longer in source) are auto-marked Stage='Archived'.

**Recommended Notion views to set up** (create once, reuse forever — Notion API can't pre-build them):

| View | Layout | Filter | Group / Sort | Use |
|---|---|---|---|---|
| **By Pile** | Board (kanban) | none | Group by Pile | Drag opps across piles, see balance |
| **Open Pipeline** | Board | Stage = Open OR Outstanding OR Discovered OR Researching | Group by Stage | Live workload |
| **Chase list** | Table | Stage = Overdue | Sort Amount desc | Money won, not collected |
| **Foundation triage** | Table | Source = Foundation, Stage = Discovered | Sort Deadline asc | Grant submissions by deadline |
| **Top 20 by $** | Table | Stage ≠ Lost AND Stage ≠ Archived | Sort Amount desc, limit 20 | Biggest moves |
| **GHL only** | Table | Source = GHL | Sort Pipeline | Mirror of GHL with click-through |

To create: open the database → click "+ Add view" → set layout → set filter → set sort → save.

### L3. Refresh cadence

Weekly cron Mon 8:15 AEST runs the page sync (`money-framework-sync` in `ecosystem.config.cjs`). Recommend adding the database sync as a parallel cron job once the database is shaped to your liking. To add: open `ecosystem.config.cjs` and copy the `money-framework-sync` block, point at `sync-opportunities-to-notion-db.mjs`, schedule `30 8 * * 1` (Mon 8:30am).
