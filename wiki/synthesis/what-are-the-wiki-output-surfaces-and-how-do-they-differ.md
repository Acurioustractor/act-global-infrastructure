# What are the wiki output surfaces and how do they differ?

> Synthesised from the wiki on 2026-04-13. This article was generated by a query and saved back into Tractorpedia so the next question can build on top of it.

## Answer

## Four Wiki Surfaces

The Tractorpedia wiki feeds four downstream surfaces, each serving a different purpose:

### 1. ACT Regenerative Studio — the public front door
- **URL:** act-regenerative-studio.vercel.app (will become act.place)
- **329 pages** across 13 sections including Sources (150)
- Card-based browser with section tabs, filtering, search
- Receives content via cross-repo GitHub Action dispatch (wiki-content-updated event)
- Includes source summaries, which Tractorpedia excludes
- **Audience:** funders, partners, public, anyone exploring ACT

### 2. Tractorpedia HTML Viewer — the encyclopedia
- **URL:** /tractorpedia.html on Command Center domain
- **189 articles** baked into a single 1.4MB self-contained HTML file
- Wikipedia Vector skin: fixed sidebar, collapsible categories, client-side search with autocomplete
- Built by wiki-build-viewer.mjs, committed by CI
- Fully offline-capable
- **Audience:** deep reading, research, reference

### 3. Command Center /wiki — the ops cockpit
- **URL:** /wiki on Command Center domain
- **189 articles** loaded live from filesystem (snapshot fallback)
- Dark glass UI, homepage is a maintenance dashboard showing repair queue: broken links, orphans, stubs, missing backlinks
- Full-text search, ReactMarkdown renderer, wikilink routing
- Reads status from wiki/output/status-latest.json
- **Audience:** Ben — internal operational use

### 4. Command Center Snapshot — legacy fallback
- 57 markdown files in apps/command-center/public/wiki/
- Labeled "Legacy mirror for command-center compatibility only"
- Only consumed as API fallback when canonical path not found
- Vestigial — canonical wiki/ now covers all paths

### Key Differences

| Surface | Articles | Source Summaries | Search | Audience |
|---------|----------|-----------------|--------|----------|
| Regen Studio | 329 | Yes (150) | Section filter | Public |
| Tractorpedia | 189 | No | Autocomplete | Deep readers |
| Command Center | 189 | No | Full-text | Internal ops |
| Snapshot | 57 | No | None | Legacy fallback |

### Sync Pipeline

Push to main triggers wiki-rebuild.yml:
1. Lint wiki
2. Update search index (OpenAI embeddings if key available)
3. Rebuild viewer + snapshot
4. Copy viewer to public/tractorpedia.html
5. Commit artifacts
6. Dispatch wiki-content-updated to act-regenerative-studio

The current act.place is a temporary holding site. When the Regen Studio moves to act.place, the 329-page wiki becomes the public knowledge layer.
