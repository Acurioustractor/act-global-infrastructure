# How does a dollar flow through the ACT ecosystem from bank transaction to R&D tax offset?

> Synthesised from the wiki on 2026-04-13. This article was generated by a query and saved back into Tractorpedia so the next question can build on top of it.

## Answer

## The Dollar Path: Bank Transaction → R&D Tax Offset

### Step 1: Bank Feed
NAB pushes transactions into Xero automatically. Every spend appears as a bank statement line.

### Step 2: Ingest
Quarterly, we export bank statement CSVs and run:
\`\`\`bash
node scripts/ingest-statement-lines-raw.mjs <file>
\`\`\`
This creates rows in the \`bank_statement_lines\` table with amount, date, payee, and direction.

### Step 3: Auto-Tag (project code assignment)
The tagging engine (\`scripts/tag-statement-lines.mjs\`) runs three rule layers:
1. **Location rules** (62 rules): spend at Palm Island → ACT-HV, spend in Mparntwe → ACT-FM
2. **Vendor rules** (26 rules): Xero subscription → ACT-IN, Hatch Electrical → ACT-PI
3. **Subscription patterns** (48 rules): recurring vendors matched by name pattern

Every line gets a project_code. The \`project_source\` field records which rule matched (location/vendor/subscription/manual).

### Step 4: Receipt Match
The matching engine (\`scripts/reconciliation-report.mjs\`) compares unmatched lines against receipt_emails using Dice-coefficient bigram similarity:
- Score ≥ 0.75: auto-apply match
- Score < 0.75: queue as candidate for human review
- Amount < $82.50 inc GST: auto-waive (ATO threshold — no tax invoice required)

### Step 5: R&D Eligibility Check
Four project codes are R&D-eligible:
- **ACT-EL** (Empathy Ledger) — Core R&D
- **ACT-IN** (Infrastructure/ALMA) — Core R&D
- **ACT-JH** (JusticeHub) — Supporting R&D
- **ACT-GD** (Goods on Country) — Supporting R&D

When a bank line is tagged to one of these codes, it automatically enters the R&D spend pool.

### Step 6: Quarterly Aggregation
The spending intelligence system aggregates by project code:
- Total eligible R&D spend = sum of all debits tagged ACT-EL + ACT-IN + ACT-JH + ACT-GD
- Receipt coverage = % of eligible spend with matched receipts
- At-risk = eligible spend missing receipts (reduces claimable offset)

### Step 7: R&D Tax Offset Calculation
\`\`\`
Eligible R&D expenditure × 43.5% = Refundable offset
Q2+Q3 FY26: $251,000 × 0.435 = $109,185
\`\`\`

### Step 8: Evidence Package
The R&D claim requires:
- **Financial evidence**: tagged bank lines + receipts (from our spending system)
- **Activity evidence**: git commits (timestamped, authored), calendar events (project-coded)
- **Technical reports**: wiki articles documenting hypothesis → experiment → outcome
- **Time records**: calendar + git history showing who worked on what, when

### The Leakage Points
Every missing receipt on an R&D-tagged transaction reduces the claimable offset:
- $100 missing receipt on ACT-EL = $43.50 lost from the refund
- Current at-risk: ~$5.3K spend = ~$2.3K offset at risk
- The receipt matching system exists specifically to minimise this leakage
