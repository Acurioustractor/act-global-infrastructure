# How does Empathy Ledger v2 integrate with CivicGraph and JusticeHub to add the human layer to the art project?

> Synthesised from the wiki on 2026-04-13. This article was generated by a query and saved back into Tractorpedia so the next question can build on top of it.

## Answer

## The Human Layer: Empathy Ledger Integration

The art project so far has network graphs, funding deserts, Hansard tracking, and animated data flows. Incredible as visualisation. Cold as argument.

The human layer is what breaks it open. Empathy Ledger v2 is already built to do this. The syndication infrastructure exists. The consent architecture exists. The API is live. The work is integration, not invention.

### What EL v2 gives us

Three API tiers, all production-ready:

**v2 API** (`empathyledger.org/api/v2/`) — key-scoped, organisation/storyteller/site auth
- Stories with markdown content, themes, images, video links
- Storytellers with bio, cultural background, avatars, elder status
- Galleries with cover images, photo counts, cultural themes
- Media assets with geo-tagged lat/lng, cultural tags, consent flags
- Transcripts with source video platform (Descript/YouTube/Vimeo)

**JusticeHub Syndication API** (`empathyledger.org/api/syndication/justicehub/`) — site-key scoped, consent-gated
- Stories, galleries, media, profiles, projects — all filtered by syndication_consent table
- Embed tokens for full story content access
- Per-gallery consent flags: allow_download, allow_embedding, allow_full_resolution
- Elder approval workflow built in

**Content Hub v1** (`/api/v1/content-hub/`) — AI-extracted curated quotes
- `/quotes?justicehub=true&project=<slug>` — returns attributed quotes by theme, impact score, storyteller

### The data model that matters

**media_assets** already has everything the art project needs:
- `cdn_url`, `thumbnail_url`, `medium_url`, `large_url` — four resolution tiers
- `latitude`, `longitude`, `location_name` — geo-tagged to the entity graph
- `cultural_tags`, `cultural_sensitivity_level` — semantic filtering
- `consent_granted`, `elder_approved`, `requires_elder_review` — consent enforced
- `project_code` — direct link to ACT project codes (ACT-OO, ACT-JH, etc.)

**storytellers** carries the human:
- `display_name`, `bio`, `cultural_background`, `is_elder`, `public_avatar_url`, `location`
- Links to stories, galleries, media, transcripts

**Consent is architecture, not policy:**
- `syndication_consent` — per-story, per-site approval with cultural_permission_level
- `gallery_syndication_consent` — per-gallery consent with elder approval fields and expiry dates
- Nothing reaches the art project without approved consent records

### How this changes the art project

Each of the 6 episodes in The Aesthetics of Asymmetry now has a human layer:

**Episode 1: The Keyhole**
- Graph side shows 587K entities, cold and abstract
- Keyhole side shows a storyteller's portrait from EL — the person the grant system doesn't see
- Hover an entity connected to an EL storyteller: their face appears, their voice plays (from transcripts.audio_url)
- The data has eyes now

**Episode 2: Resolution**
- Blur reveals data. But it also reveals faces.
- As resolution sharpens on the network, storyteller portraits emerge from the clusters
- You can see who is inside each node. The node is not a number. It is a person with a story.
- Pull `/api/v2/media?type=image` filtered by `cultural_sensitivity_level=public`

**Episode 3: The Missing Folder**
- The 873 funding deserts. Dark voids on the map.
- But EL storytellers live in those deserts. Laquisha in Mparntwe. Uncle Allan on Palm Island. Nigel in Double Camp.
- Each desert tile, when scrolled over, reveals the voices of people who live there
- What the government didn't measure, EL captured with consent
- Pull `/api/syndication/justicehub/media` filtered by lat/lng bounding box per LGA

**Episode 4: The Promise and the Purchase Order**
- Hansard quote floats above: "$50 million for youth justice diversion"
- Purchase order data flows below, showing where the money actually went
- EL storyteller appears between them: Jackquann, 14, saying "Detention. That's not my home."
- The gap is no longer abstract. It has a face and a voice.
- Pull `/api/v1/content-hub/quotes?theme=youth-justice&justicehub=true`

**Episode 5: The Alibi**
- The chaos reveals itself. Everything at once.
- But storytellers are permanent. They don't drown in the data.
- Each EL profile becomes an anchor point — a place the viewer can ground themselves
- Choose a storyteller instead of a postcode. See the data from their life outward.
- Pull `/api/v2/storytellers` with cross-reference to entity graph

**Episode 6: CARE**
- Data recedes. Storytellers centre.
- OCAP principles rendered as architecture: who owns, who controls, who accesses, who possesses
- Every story visible on the site exists because consent was given, and consent can be revoked
- The `gallery_syndication_consent` table becomes visible infrastructure — a consent register as public art
- Pull `/api/syndication/justicehub/galleries` with elder_approved filter

### The technical integration

**Phase 1: Register the art project as a syndication site**

```sql
INSERT INTO syndication_sites (
  slug, name, api_base_url, allowed_domains, site_type
) VALUES (
  'aesthetics-of-asymmetry',
  'The Aesthetics of Asymmetry',
  'https://aesthetics.civicgraph.app',
  ARRAY['aesthetics.civicgraph.app', 'civicgraph.app'],
  'art-project'
);
```

Then issue a site-scoped API key.

**Phase 2: Approve consent for featured storytellers**

For the 12 Oonchiumpa storytellers identified earlier (Kristy, Tanya, Laquisha, Nigel, Patricia, Aunty Bev, the Bloomfield family, Braydon), run:

```sql
INSERT INTO syndication_consent (
  story_id, site_id, storyteller_id, status, cultural_permission_level
) SELECT s.id, (SELECT id FROM syndication_sites WHERE slug='aesthetics-of-asymmetry'),
  s.storyteller_id, 'approved', 'public'
FROM stories s
WHERE s.storyteller_id IN (...);
```

Same pattern for galleries via `gallery_syndication_consent`.

**Phase 3: Build the integration layer**

A single API route on the art project side:

```typescript
// /api/human-layer/[entityId]
// Given a CivicGraph entity ID, return:
// - All EL storytellers linked via project_code
// - Their consented photos, quotes, transcripts
// - Geo-tagged media within N km
```

Calls EL syndication API, caches in Redis, serves to the visualisation layer.

**Phase 4: Visual treatment of human data**

Three modes:
- **Portrait anchors** — storyteller faces appear on entity hover
- **Voice pins** — transcript audio plays when a region is focused
- **Quote overlays** — curated quotes float above data when hovering over themes

All driven by the same data pipeline — the `project_code` field on `media_assets` is the join key across CivicGraph entities, JusticeHub programs, and Empathy Ledger storytellers.

It's about proving the entire thesis of the art project: that systemic data alone cannot shift power, that narrative alone cannot prove patterns, and that the Third Reality emerges only when both layers are held together in the same frame.
