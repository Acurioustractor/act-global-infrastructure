A Curious Tractor

How LLMs Turn Raw Research
Into a Living Knowledge Base

Collect sources. Let the LLM help structure them. Ask better questions. Durable answers file back into the wiki.

Core Insight
Humans steer the wiki. The LLM helps compile, connect, and maintain it. Every durable answer compounds.
Main Pipeline
Step 01
📥 Collect Sources
Dump articles, papers, websites, conversations, Notion pages, and images into a raw folder. Use Obsidian Web Clipper for articles.
  • Website scrapes (JusticeHub, PICC, etc.)
  • Notion databases and pages
  • Gemini/Claude research conversations
  • Government data (AusTender, ACNC)
Step 02
🤖 LLM Compiles Wiki
The LLM reads what lands in raw/ and helps build a structured wiki of .md files — summaries, concept articles, and links between related ideas across sources.
  • Auto-generated index file
  • Projects, concepts, communities, research, decisions
  • [[Wikilinks]] between related topics
  • Backlinks for reverse navigation
Step 03
🔍 Ask Questions
Query the wiki with complex questions. The LLM reads the index, follows links, and finds answers across the collection without a heavyweight retrieval stack.
  • Cross-domain synthesis via index + backlinks
  • Canonical graph plus synthesis layer
  • Multi-step research queries
  • /wiki query "what connects PICC to Third Reality?"
Index
Organize
Query · Render · File Back
Step 04
📊 Render Outputs
Outputs go beyond text — the LLM renders markdown files, Marp slide decks, data visualizations, and Tractorpedia pages. All viewable in Obsidian or browser.
  • Tractorpedia (Wikipedia-style viewer)
  • Markdown articles for Obsidian
  • Grant narratives and reports
  • R&D tax evidence packages
Step 05
🔄 File Back to Wiki
Every output gets filed back into the wiki. Every question you ask makes the knowledge base smarter. The loop compounds — your research builds on itself.
  • Outputs go back to wiki/
  • Knowledge compounds over time
  • Grant work enriches project articles
  • The loop never stops
🔁
Filed Back — Knowledge Compounds
This is the real insight. The wiki grows from both sides: what you save and what you ask. It is not a self-writing brain. It is a compounding one: humans curate and decide what matters, the LLM helps turn that into durable memory.
Support Layer
📓
Obsidian (Backend)
View raw data, the compiled wiki, and visualizations all in one place. The team edits and explores here. Agents compile against the same structure. Graph view shows the connections.
🏛️
Tractorpedia (Frontend)
Wikipedia-style visual browser for the wiki. Navigate articles, search, explore backlinks. Shareable — anyone can browse the knowledge base.
🔧
Lint + Heal
Health checks find broken links, scope drift, stale copy, and missing connections. The system improves when humans review those signals and the agent patches the graph.
